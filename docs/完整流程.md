## 1. ⚡ 快速创作 (Quick Create / Standard)

**功能概述：** 输入主题或文案，由 AI 辅助生成分镜、旁白和画面，最终合成完整视频。

### 第一板块：创作模式与文案输入

| 字段 | 类型 | 说明 |
|------|------|------|
| 创作模式 | 单选按钮 | `AI 生成分镜` / `固定文案` |
| 视频标题 | 文本输入 | 可选，留空由 AI 自动生成 |
| 主题 / 创作方向 | 多行文本 | 生成模式下输入主题/卖点/风格/营销方向 |
| 固定旁白文案 | 多行文本 | 固定文案模式下输入完整旁白脚本 |
| 分镜数量 | 滑块 (1~20) | 生成模式专属，AI 自动拆分场景数量 |
| 视频帧率 | 滑块 (15~60fps) | 默认 30fps |

**逻辑说明：**
- **AI 生成分镜模式：** 输入主题后，AI 自动拆分场景、生成旁白和画面提示词
- **固定文案模式：** 直接使用输入内容作为旁白脚本，按原文组织画面

### 第二板块：画面与媒体配置

| 字段 | 类型 | 说明 |
|------|------|------|
| 画面模板 | 下拉选择 | 从可用模板中选择（支持横/竖屏，如 1080x1920, 1920x1080） |
| 媒体工作流 | 下拉选择 | 选择图片/视频生成工作流（可选） |
| 图片风格前缀 | 多行文本 | 在生成图片提示词前添加固定前缀 |

**模板来源：** 预置在 `templates/` 目录下，按分辨率存放（1080x1080, 1080x1920, 1920x1080）。

### 第三板块：🎤 配音合成 (TTS)

**TTS 模式选择：** `本地` / `ComfyUI`

#### 3.1 本地合成

##### 3.1.1 Edge TTS（本地默认）

| 字段 | 类型 | 说明 |
|------|------|------|
| 音色选择 | 下拉列表 | 从预置的 Edge TTS 音色列表选择（支持中英文等多语言） |
| 语速 | 滑块 (0.5x ~ 2.0x) | 默认 1.2x |

##### 3.1.2 VoxCPM API（在线）

| 字段 | 类型 | 说明 |
|------|------|------|
| CFG 强度 | 滑块 (0.5~5.0) | 控制生成语音的风格强度，默认 2.0 |
| 归一化 Normalize | 复选框 | 是否对音频进行归一化处理 |
| 降噪 Denoise | 复选框 | 是否对音频进行降噪处理 |
| 控制指令 | 文本输入 | 描述语音风格，如"自然、温柔、带一点营销感" |
| 启用 Prompt Text | 复选框 | 是否启用极致克隆模式的参考文本 |
| Prompt Text | 多行文本 | 极致克隆模式的参考文本（上传参考音频后可使用**语音转文字**按钮） |
| 参考音频 | 上传组件 | 📤 上传新文件 或 📂 从历史选择 |

**极致克隆模式流程：**
1. 勾选"启用 Prompt Text"复选框
2. 选择参考音频（上传或从历史选择）
3. 点击右侧 🎙️ 按钮，自动将参考音频内容转换为文字填入文本框
4. 系统将此文字作为克隆参考传递给 VoxCPM API

#### 3.2 ComfyUI 合成

| 字段 | 类型 | 说明 |
|------|------|------|
| TTS 工作流 | 下拉选择 | 从已配置的 ComfyUI TTS 工作流中选择 |
| 参考音频（可选） | 上传组件 | 📤 上传新文件 或 📂 从历史选择 |

### 第四板块：背景音乐

| 字段 | 类型 | 说明 |
|------|------|------|
| 背景音乐 | 下拉选择 | 从 BGM 文件列表中选择，或选择"不使用 BGM" |
| BGM 音量 | 滑块 (0~1.0) | 背景音乐音量，步长 0.05 |

### 第五板块：🎬 生成视频

**前置校验：**
- 主题/文案不能为空
- 画面模板必须已选择

**生成流程：**
1. 用户点击"开始生成"按钮
2. 系统提交任务到 FastAPI 后端 `/api/video/generate/async`
3. 开始轮询任务状态（每 3 秒）
4. 进度条实时更新
5. 生成完成后显示视频预览
6. 支持下载生成的 MP4 文件

## 2. 🎨 自定义素材 (Asset Based / Custom Media)

**功能概述：** 上传自定义图片/视频素材，通过 AI 分析后生成合成视频。

### 第一板块：素材上传

| 字段 | 类型 | 说明 |
|------|------|------|
| 图片素材 | 上传组件 | 支持 jpg/jpeg/png/gif/webp，可多选 |
| 视频素材 | 上传组件 | 支持 mp4/mov/avi/mkv/webm，可多选 |

每个素材上传区域均支持：
- 📤 **上传新文件：** 点击或拖拽上传
- 📂 **从历史选择：** 从本地历史记录中选择已上传的文件

### 第二板块：视频信息

| 字段 | 类型 | 说明 |
|------|------|------|
| 视频标题 | 文本输入 | 如"宠物店年终促销" |
| 创作意图 | 多行文本 | 描述卖点、风格、受众等 |
| 目标时长 | 滑块 (15~120秒) | 最终视频的目标时长 |

### 第三板块：⚙️ 素材分析与动画配置

#### 3.1 素材分析服务

| 字段 | 类型 | 说明 |
|------|------|------|
| 素材分析来源 | 单选按钮 | ☁️ RunningHub（云端） / 本地 ComfyUI / API VLM |

**选择逻辑：**
- **RunningHub（云端）：** 需要配置 `runninghub_api_key`，使用 `analyse_image.json` / `analyse_video.json` 工作流
- **本地 ComfyUI：** 需要配置 `comfyui_url`，使用 `selfhost/` 下的分析工作流
- **API VLM：** 使用已配置的 DashScope/OpenAI/Gemini VLM 模型分析素材

| 字段 | 类型 | 说明 |
|------|------|------|
| 分析图片工作流 | 文本输入 | 可自定义，如 `runninghub/analyse_image.json` |
| 分析视频工作流 | 文本输入 | 可自定义，如 `runninghub/analyse_video.json` |
| API VLM 模型 | 文本输入 | API 来源时填写模型 key |

#### 3.2 素材动画服务

| 字段 | 类型 | 说明 |
|------|------|------|
| 素材动画服务 | 单选按钮 | `不启用` / `API 调用` |

选择 API 后，可配置图生视频的动画工作流和参数（时长、比例等）。

### 第四板块：🎤 配音合成

| 字段 | 类型 | 说明 |
|------|------|------|
| TTS 声音 | 下拉选择 | 从预置 Edge TTS 音色中选择 |
| TTS 语速 | 滑块 (0.5x ~ 2.0x) | 默认 1.2x |

### 第五板块：🎵 背景音乐

| 字段 | 类型 | 说明 |
|------|------|------|
| 背景音乐 | 下拉选择 | 从 BGM 文件列表中选择或"不使用 BGM" |
| BGM 音量 | 滑块 (0~1.0) | 步长 0.05 |
| BGM 模式 | 隐藏参数 | 默认 `loop` 循环播放 |

### 第六板块：🎬 生成视频

**前置校验：**
- 必须上传至少一个素材（图片或视频）

**生成流程：**
1. 分析素材内容（根据选择的来源调用工作流/VLM）
2. AI 生成脚本和分镜
3. 逐帧处理素材
4. TTS 配音合成
5. 视频拼接与输出
6. 预览和下载

## 3. 🤖 数字人口播 (Digital Human)

**功能概述：** 上传人物形象和商品图片，结合 TTS 配音，生成数字人口播视频。

### 第一板块：人物形象上传

| 字段 | 类型 | 说明 |
|------|------|------|
| 角色图片 | 上传组件 | 📤 上传新文件 或 📂 从历史选择（支持 jpg/jpeg/png/webp，可多选） |

### 第二板块：🎤 配音合成

**TTS 模式选择：** `本地` / `ComfyUI`

#### 2.1 本地合成

##### Edge TTS

| 字段 | 类型 | 说明 |
|------|------|------|
| 音色选择 | 下拉列表 | 从预置 Edge TTS 音色中选择 |
| 语速 | 滑块 (0.5x ~ 2.0x) | 默认 1.2x |

##### VoxCPM API（在线）

| 字段 | 类型 | 说明 |
|------|------|------|
| CFG 强度 | 滑块 (1.0~5.0) | 默认 2.0，控制语音风格强度 |
| 归一化 Normalize | 复选框 | 音频归一化 |
| 降噪 Denoise | 复选框 | 音频降噪 |
| 控制指令 | 文本输入 | 语音风格指令，如"自然、温柔" |
| 启用 Prompt Text | 复选框 | 是否启用参考文本 |
| Prompt Text | 文本输入 | 参考音频的文字内容，右侧 🎙️ 按钮可自动转写 |

**参考音频上传区：**
- 📤 上传新文件（支持 mp3/wav/flac/m4a/aac/ogg）
- 📂 从历史选择

**提示文本功能：**
- 勾选"启用 Prompt Text"后显示输入框
- 输入框右侧有 🎙️ 按钮
- 点击后自动调用 DashScope ASR 服务将参考音频转写为文字填充到输入框

#### 2.2 ComfyUI 合成

| 字段 | 类型 | 说明 |
|------|------|------|
| TTS 工作流 | 下拉选择 | 从已配置的 ComfyUI TTS 工作流中选择 |
| 参考音频 | 上传组件 | 📤 上传新文件 或 📂 从历史选择 |

### 第三板块：⚙️ 服务配置

#### 3.1 前置图片生成服务

两个选择性：

| 选项 | 说明 |
|------|------|
| ☁️ RunningHub（云端） | 使用 `digital_image.json` 前置工作流，需要配置 RunningHub API Key |
| 本地 ComfyUI | 使用 `selfhost/digital_image.json` |
| API 模型 | 调用已配置的图像 API 模型 |

**对应下拉选择：** 前置图片工作流/模型

#### 3.2 口播视频合成服务

两个选择性：

| 选项 | 说明 |
|------|------|
| ☁️ RunningHub（云端） | 使用 `digital_combination.json` 口播合成工作流 |
| 本地 ComfyUI | 使用 `selfhost/digital_combination.json` |
| API 模型 | 调用已验证的 API 口播视频模型（需支持 digital_human 能力） |

**对应下拉选择：** 口播视频工作流/模型

**各工作流路径：**
- 前置图片工作流：`workflows/runninghub/digital_image.json`
- 商品合成图片工作流：`workflows/runninghub/digital_customize.json`
- 口播视频合成工作流：`workflows/runninghub/digital_combination.json`
- API 图片工作流：可选 `api/...`
- API 口播视频工作流：可选 `api/...`

### 第四板块：💫 选择生成模式

#### 💻 带货模式 (digital)

| 字段 | 类型 | 说明 |
|------|------|------|
| 商品图片 | 上传组件 | 📤 上传新文件 或 📂 从历史选择（可多选） |
| 口播文案输入框 | 多行文本 | 可填写固定口播文案；留空时 AI 自动根据商品标题生成 |
| AI 创作旁白输入框 | 隐藏字段 | 商品标题作为 AI 生成文案的参考（`goods_title`） |
| TTS 声音 | 文本输入 | 选择 TTS 音色 ID |
| TTS 语速 | 滑块 (0.5x~2.0x) | 默认 1.2x |

**带货模式生成流程（RunningHub/ComfyUI）：**
1. 第一步（前置图片）：用 `digital_image.json` 工作流将角色图 + 商品图合成为数字人形象图
2. 第二步（TTS 配音）：根据文案生成配音音频
3. 第三步（视频合成）：用 `digital_combination.json` 工作流将合成图 + 音频合成为口播视频

**带货模式生成流程（API 模式）：**
1. 第一步（图片生成）：API 调用图像模型生成数字人商品推广图
2. 第二步（文案生成）：AI LLM 根据商品标题自动生成推广文案（如未提供）
3. 第三步（TTS 配音）：根据文案生成配音音频
4. 第四步（视频合成）：API 调用口播视频模型生成最终视频

#### 🧐 自定义模式 (customize)

| 字段 | 类型 | 说明 |
|------|------|------|
| 自定义文本输入框 | 多行文本 | 填写固定口播文案内容 |

**自定义模式生成流程：**
1. 使用角色图片作为视频画面
2. TTS 配音：将自定义文本合成为音频
3. 口播视频合成：用 `digital_combination.json` 工作流将角色图 + 音频合成为口播视频

### 第五板块：🎬 生成视频

**前置校验：**
- 必须上传角色图片
- 带货模式：必须上传商品图片
- 带货模式：必须有口播文案或商品标题（至少一项）
- 自定义模式：必须填写自定义文本

**校验失败时：**
- 显示对应提示信息
- "生成视频"按钮置灰不可用

**校验通过后：**
- 显示提示信息
- "生成视频"按钮可用

**生成流程：**
1. 验证系统配置（LLM + ComfyUI）
2. 创建任务输出目录
3. 根据模式执行对应工作流链
4. 进度条跟踪（图片合成 → TTS → 视频合成）
5. 下载生成的 MP4 文件
6. 自动保存生成历史到本地

## 4. 🎥 图生视频 (Image to Video)

**功能概述：** 上传首帧图片，通过提示词驱动视频生成。

### 第一板块：首帧图片上传

| 字段 | 类型 | 说明 |
|------|------|------|
| 首帧图片 | 上传组件 | 📤 上传新文件 或 📂 从历史选择 |

### 第二板块：提示词输入

| 字段 | 类型 | 说明 |
|------|------|------|
| 提示词 | 多行文本 | 描述画面如何运动、镜头、风格等（如"镜头缓慢推进，阳光洒落"） |

### 第三板块：🎥 工作流选择

| 字段 | 类型 | 说明 |
|------|------|------|
| 图生视频工作流 | 下拉选择（可创建） | 自动过滤 `i2v_` 前缀或 `api/` 开头的工作流 |
| API 参数 JSON（可选） | 多行文本 | 如 `{"duration":5,"video_ratio":"9:16"}` |

**工作流过滤规则：**
- 从 `mediaWorkflows` 列表中自动筛选
- 匹配 `key.startsWith('api/')` 或 `name.startsWith('i2v_')` 或 `key.includes('/i2v_')`
- 支持手动输入自定义工作流 key

### 第四板块：🎬 生成视频

**前置校验：**
- 必须上传首帧图片
- 提示词不能为空
- 必须选择图生视频工作流

**提交接口：** POST `/api/pipelines/image-to-video/async`

**请求参数：**
```json
{
  "image_assets": ["path/to/image.png"],
  "prompt_text": "描述画面运动的提示词",
  "workflow_key": "i2v_workflow_key",
  "api_video_params": {"duration": 5, "video_ratio": "9:16"}
}
```

## 通用模块说明

### 上传与历史选择机制

所有流程中的上传组件均统一使用 `UploadBox` 组件，提供两种选择方式：

1. **📤 上传新文件：** 拖拽或点击选择文件，自动上传到服务器 `temp/uploads/` 目录
2. **📂 从历史选择：** 弹出 `HistoryDialog` 对话框，显示本地历史记录（最近 50 条）

**上传分类：**
- `image` - 图片素材
- `video` - 视频素材
- `ref_audio` - 参考音频
- `character_image` - 数字人角色图
- `goods_image` - 数字人商品图

### 任务中心

所有流程通过统一的任务中心管理生成任务：

- **提交：** POST 到对应 API 端点
- **轮询：** 每 3 秒查询任务状态 `/api/tasks/{task_id}`
- **状态：** pending → running → completed / failed / cancelled
- **结果：** 完成后显示视频预览和下载按钮

### 任务输出目录

所有生成结果保存在 `tasks/output/{task_id}/` 目录下，按任务 ID 区分。
